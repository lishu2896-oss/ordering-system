# 金玉越南料理牛肉河粉 — 餐廳點餐系統規格書

## 一、專案概述

**餐廳名稱：** 金玉越南料理牛肉河粉（Kim Ngọc món ăn Việt Nam）  
**系統目標：** 建立一套涵蓋「顧客點餐 → 櫃台結帳 → 廚房收單」完整流程的單頁網頁應用程式（SPA）。  
**技術限制：** 純前端 HTML + CSS + JavaScript（單一 .html 檔案），無需後端，使用 localStorage 模擬跨畫面資料傳遞。  
**視覺風格：** 越南風格 — 以紅色（#C0392B）、金黃色（#F1C40F）、米白色（#FDF6E3）為主色調，搭配越南傳統裝飾花紋（SVG 或 CSS border pattern），字型使用 Google Fonts 的 Playfair Display 或 Noto Serif TC，整體呈現街邊小館的溫暖感。

## 三、系統架構

### 3.1 三大介面（View）

系統以單一 HTML 檔案實作，透過 JavaScript 控制顯示/隱藏切換三個主要 View：

| View ID | 名稱 | 使用者 | 進入方式 |
|---|---|---|---|
| `view-customer` | 顧客點餐畫面 | 顧客 | 首頁預設畫面（掃 QR Code 後進入） |
| `view-cashier` | 櫃台結帳畫面 | 店員 | 頁面右上角隱藏連結 `/cashier` 或 `?view=cashier` |
| `view-kitchen` | 廚師收單畫面 | 廚師 | 頁面右上角隱藏連結 `?view=kitchen` |

---

## 四、介面一：顧客點餐畫面（Customer View）

### 4.1 流程

```
Step 1：選擇桌號
    ↓
Step 2：瀏覽菜單 & 加入購物車
    ↓
Step 3：確認訂單
    ↓
Step 4：送出（顯示「請至櫃台結帳」提示）
```

### 4.2 Step 1 — 選擇桌號

- 畫面中央顯示餐廳 Logo（金玉）及越南裝飾紋樣
- 大標題：「請選擇桌號 / Chọn số bàn」
- 顯示桌號按鈕：1 ～ 15，以 3×5 網格排列
- 顧客點選桌號後，以高亮顯示選中狀態
- 點選「開始點餐 / Bắt đầu gọi món」按鈕進入 Step 2
- 若未選桌號直接送出，顯示提示「請先選擇桌號」

### 4.3 Step 2 — 點餐主畫面

**頂部導覽：**
- 左：餐廳名稱（金玉 Kim Ngọc）
- 中：桌號顯示（桌號 #X）
- 右：購物車圖示 + 品項數量 badge

**分類標籤列（Tabs / 橫向捲動）：**
- 顯示所有分類，以越南風格底線 Tab 呈現
- 週六週日限定分類需標示「限定 ★」

**菜單品項卡（Card List）：**
- 每個品項顯示：
  - 中文名稱（主要，大字）
  - 越南文名稱（副標，小字，斜體）
  - 價格（TWD，紅色金額字）
  - `＋` 加入按鈕（點擊後按鈕短暫動畫反饋）
  - 若已加入購物車，顯示數量調整器（`－ N ＋`）

**浮動購物車底部列：**
- 固定在畫面底部
- 顯示：已選 N 項 ／ 小計 NT$ XXX
- 點擊展開訂單確認彈窗（Step 3）

### 4.4 Step 3 — 確認訂單彈窗（Modal）

- 以全屏半透明遮罩 + 白色卡片顯示
- 列出所有已選品項：名稱、數量、小計
- 支援在此刪除或調整數量
- 顯示合計金額（NT$）
- 備註欄位（選填，placeholder：「特殊需求、忌口…」）
- 內用 / 外帶 切換（預設內用）
- 按鈕：「繼續點餐」／「確認送出」

### 4.5 Step 4 — 送出成功畫面

- 顯示越南風格大圖示（碗+筷子）
- 文字：「訂單已收到！請至櫃台結帳 🧾」
- 顯示：桌號、訂單編號（自動產生，格式 `KN-YYYYMMDD-XXXX`）、品項列表、總金額
- 按鈕：「返回首頁（重新點餐）」
- 訂單資料寫入 `localStorage` key: `orders`，狀態設為 `pending_payment`

## 五、介面二：櫃台結帳畫面（Cashier View）

### 5.1 畫面佈局

- 左側列表（60%）：所有待結帳訂單
- 右側詳情（40%）：選中訂單的詳細內容

### 5.2 訂單列表

- 每筆訂單卡片顯示：
  - 訂單編號
  - 桌號
  - 下單時間（HH:MM）
  - 品項數量
  - 總金額
  - 狀態標籤：`待結帳`（黃）/ `已付款待出餐`（藍）/ `完成`（綠）
- 依下單時間排序（最新在上）
- 可篩選狀態（全部 / 待結帳 / 進行中）

### 5.3 訂單詳情面板

點選訂單後右側顯示：
- 訂單編號、桌號、下單時間
- 品項明細（名稱 × 數量 = 小計）
- 備註（若有）
- 內用 / 外帶
- **合計金額（大字顯示）**
- 付款方式選擇：現金 / 行動支付（UI 切換，不實作真實支付）
- **「確認付款完成」按鈕**（紅色主按鈕）
  - 點擊後彈出確認 Dialog：「確認收款 NT$ XXX？」
  - 確認後：訂單狀態改為 `paid`，同時新增到廚房收單列表
  - 顯示成功提示 Toast：「✅ 已確認付款，訂單傳送至廚房」

### 5.4 統計列（頁面頂部）

- 今日訂單數：N 筆
- 今日營業額：NT$ XXXXX
- 待處理：N 筆

---

## 六、介面三：廚師收單畫面（Kitchen View）

### 6.1 畫面設計

- 深色背景（#1A1A1A），高對比白字，適合廚房環境閱讀
- 訂單以大卡片方式顯示（字體放大至少 16px）
- 頁面自動每 10 秒刷新一次（從 localStorage 讀取最新資料）

### 6.2 訂單卡片

每張卡片包含：
- **桌號**（超大字，顯眼）
- 訂單編號（小字）
- 下單時間 + 已等待分鐘數（例：「已等待 8 分鐘」，超過 15 分鐘變紅色警示）
- 品項列表（每項一行，大字清楚顯示）：
  - 品項中文名稱 × 數量
- 備註（若有，以橘色高亮顯示）
- 內用 / 外帶 標籤
- **「完成出餐 ✓」按鈕**（綠色大按鈕）
  - 點擊後訂單狀態改為 `completed`，卡片從列表移除

### 6.3 排序與分欄

- 依等待時間排序（等最久的在最左/最上）
- 支援 2 欄顯示（桌面）或 1 欄（手機）

### 6.4 狀態欄

頁面頂部顯示：
- 目前待製作訂單：N 筆
- 最後更新時間

## 十一、開發注意事項

1. **全部實作在單一 HTML 檔案中**（含 CSS、JavaScript）
2. **使用 localStorage 模擬跨 View 資料同步**（實際部署時可換成 Firebase Realtime Database）
3. **廚師畫面每 10 秒 `setInterval` 從 localStorage 重新讀取資料**，模擬即時更新
4. **不需要後端 API、不需要登入驗證**
5. 所有金額單位為新台幣（NT$）
6. 日期/時間使用台灣時區（Asia/Taipei）
7. Google Fonts 從 CDN 引入：`https://fonts.googleapis.com/css2?family=Playfair+Display&family=Noto+Serif+TC&display=swap`
